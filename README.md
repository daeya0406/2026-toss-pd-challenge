# 일정 맞추기 — 토스 프로덕트 디자이너 챌린지 2026

6명 회의 일정을 **겹쳐 보고** 모두 괜찮은 시간을 찾는 모바일 프로토타입입니다.

## 시작하기

**👉 [일정 맞추기](index.html)** — 회의 만들기 · 일정 입력 · 가능한 시간

| 구분 | 링크 |
|------|------|
| 프로토타입 허브 | `index.html` |
| UI 컴포넌트 | `components.html` |
| 제출 답변 원문 | `제출문항.md` |

> Figma 없이 **웹 프로토타입만** 제출합니다. `index.html`에서 플로우별 화면으로 들어갈 수 있습니다.

## 둘러보기 순서

1. [문제 정의](docs/view.html?doc=problem) (30초)
2. [UI Components](components.html) (45초)
3. [회의 만들기](create.html) → [일정 입력](respond.html?screen=2) → [히트맵](merge.html?screen=2) (90초)
4. [개별 일정 겹쳐보기](merge.html?screen=6) — 차별점 (45초)
5. [설계 결정](docs/view.html?doc=decisions) (30초)

## 배포

[docs/view.html?doc=submit](docs/view.html?doc=submit)에 GitHub Pages / Vercel / Netlify 절차가 정리되어 있습니다.

배포 후 `제출문항.md`의 **배포 URL** 칸에 실제 주소를 넣으세요.

## 구조

```
index.html       ← 제출 시작점 (허브)
create.html       ← 주최자: 회의 만들기
respond.html    ← 참석자: 일정 입력
merge.html        ← 주최자: 히트맵 · 확정
components.html  ← UI 참고
docs/            ← 문제 · 프로세스 · 설계 결정
```

자세한 파일 설명은 [docs/view.html?doc=structure](docs/view.html?doc=structure).
