문제 정의

6명이 다음 주 안에 1시간 회의를 잡을 때, 역할별로 어떤 문제가 생기는지 정리했습니다.

주최자 — 정보가 흩어져 있다

불가·비선호·필수/선택이 채팅·구두·캘린더에 흩어져 있어, “모두 되는 시간”을 찾다 한 명씩 다시 확인하는 순환 조율이 반복됩니다.

병목은 수집이 아니라, 받은 조건을 한 화면에서 비교·결정하지 못하는 것입니다.

참석자 — 입력 부담

사유까지 적으라 하면 미루거나 포기합니다. 실제로는 “언제 안 되는지”만 알려주고 맥락은 채팅으로 전달합니다.

조건의 영향력이 다른데 한 줄로 취급

조건영향
필수 3명 모두 가능바로 확정 후보
필수 1명 불가해당 시간 후보 제외
선택 1명 불가조율 대상 (회의는 열 수 있음)

설계 결정 (핵심)

  • 역할 분리 — 주최자(create·merge) / 참석자(respond) 플로우 분리
  • 언제만 입력 — 불가·비선호만 받고 사유는 조율(채팅)로
  • 요일 → 시간 — 외근은 요일 전체 불가 후 시간 그리드
  • 필수/선택 — 100% 합의 없이 확정 가능
  • 히트맵 — 전원 가능은 바로 확정, 나머지는 이름 뱃지로 조율 대상 표시
  • 일정 겹쳐보기 — 이름 토글로 레이어 합성 (merge screen 6)

→ 일정 겹쳐보기 화면 열기

설계 프로세스

7단계 프레임워크(목표 → 오디언스 → 컨텍스트 → 아이디어 → 플로우 → UI → 검증)에 맞춰 정리했습니다.

핵심 목표: 6명 회의 조율에서 조건의 영향력 차이를 반영해, 주최자는 빠르게 시간을 정하고 참석자는 부담 없이 입력하게 한다.

히트맵 규칙

  • 필수 참석자가 요일 전체 불가 → 해당 요일 열 전체 회색
  • 선택 참석자만 요일 불가 → 열은 유지, 해당 인원만 전 시간 불가
  • 전원 가능 칸 → 「전원 가능」
  • 필수 비선호 → 이름 뱃지에 * 표시

→ 히트맵 화면 열기
