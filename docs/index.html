문제 정의

처음 아이디어는 6명 일정을 같은 주간 그리드에 겹쳐 보는 것이었다. 각자 일정은 있는데 한 타임테이블에 겹쳐 보이지 않으면, 주최자는 머릿속으로 합치거나 한 명씩 다시 물어봐야 한다.

기존 방식의 한계

캘린더·일정 투표는 가능 시간만 표로 모을 뿐, 같은 칸에 어떻게 겹치는지 한눈에 읽기 어렵다.

설계에서 정한 것 · 겹치면서 나온 경우의 수

겹치기만 하면 끝이 아니라, 칸마다 어떻게 보여줄지 규칙이 필요했다. 필수/선택에 따라 같은 불가도 다르게 읽히고, 요일 전체 불가와 시간 칸 불가를 나눴으며, 애매한 칸은 가능한 시간에서 비우고 일정 겹쳐보기로 넘긴다.

겹친 조합 (예)화면
전원 가능파랑 · 전원 가능
필수 1명 불가회색 · 후보 제외
선택 1명 불가이름 뱃지 · 조율

설계 결정 (핵심)

  • 역할 분리 — 주최자(create·merge) / 참석자(respond) 플로우 분리
  • 언제만 입력 — 불가·비선호만 받고 사유는 조율(채팅)로
  • 요일 → 시간 — 외근은 요일 전체 불가 후 시간 그리드
  • 필수/선택 — 100% 합의 없이 확정 가능
  • 가능한 시간 — 참여율 높은 시간 그리드 · 전원 가능 / 이름 뱃지
  • 일정 겹쳐보기 — 이름 토글로 레이어 확인 (merge screen 6)

설계 프로세스

7단계 프레임워크(목표 → 오디언스 → 컨텍스트 → 아이디어 → 플로우 → UI → 검증)에 맞춰 정리했습니다.

핵심 목표: 6명 일정을 같은 주간 그리드에 겹쳐 보이게 한다.

가능한 시간 · 규칙

  • 필수 참석자가 요일 전체 불가 → 해당 요일 열 전체 회색
  • 선택 참석자만 요일 불가 → 열은 유지, 해당 인원만 전 시간 불가
  • 전원 가능 칸 → 「전원 가능」
  • 필수 비선호 → 이름 뱃지에 * 표시
