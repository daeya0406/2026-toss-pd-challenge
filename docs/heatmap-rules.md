# 히트맵 · 슬롯 경우의 수

주최자 히트맵(`merge.html` screen 2)과 일정 겹쳐보기(screen 6)에서 **조건 조합 → 표시** 규칙입니다.  
구현: `js/mock-data.js` · `classifyHeatSlot` · `heatDisplayPills`

---

## 1. 요일 전체 불가 (외근·재택)

참석자가 respond 3단계에서 「요일 전체 불가」를 선택한 경우.

| 누가 | 히트맵 열 | 슬롯 단위 | 회의 후보 |
|------|-----------|-----------|-----------|
| **필수** 인원 | **열 전체 회색** (선택·탐색 불가) | 해당 인원 전 시간 `unavail` | 그 요일 **후보 제외** |
| **선택** 인원 | 열은 일반 (회색 아님) | 해당 인원만 전 시간 `unavail` | **후보 유지** · 이름 뱃지로 조율 |

**이유:** 선택 인원은 불참해도 회의를 열 수 있어야 함.

### mock 예시 (1주차)

| 요일 | 누가 | 역할 | 화면 |
|------|------|------|------|
| 월 | 최수진 | 선택 | 월요일 열 정상 · 칸마다 최수진 불가 반영 |
| 화 | 김민수 | 필수 | **화요일 열 전체 회색** |
| 금 | 김민수 | 필수 | **금요일 열 전체 회색** |

## 2. 시간 슬롯 단위 (그리드 입력)

| 조건 | 히트맵 칸 | 확정 |
|------|-----------|------|
| 제출자 전원 `ok` | 「전원 가능」 (파랑) | 바로 확정 후보 |
| 필수 1명+ `unavail` | 회색 빈 칸 (후보 제외) | — |
| 선택만 `unavail` | 이름 1명 (빨강 뱃지) | 조율 후 확정 가능 |
| 필수만 `pref` | 이름 최대 2명 (노랑) | 조율 우선 |
| 선택만 `pref` | 이름 최대 2명 | 참고 |
| 불가+비선호 혼재 | 빈 칸 | 일정 겹쳐보기로 확인 |
| 비선호 3명+ | 빈 칸 | 일정 겹쳐보기로 확인 |
| ok &lt; 3명 (제출 기준) | 회색 빈 칸 | 후보 제외 |

---

## 3. 역할 · 상태 조합 (요약)

```
요일 전체 불가?
├─ 필수 → 열 회색
└─ 선택 → 그 사람만 unavail, 열은 살아 있음

시간 칸?
├─ 필수 unavail → 칸 제외
├─ 선택 unavail → 이름 1명 (조율)
├─ pref만 → 이름 ≤2명
└─ 전원 ok → 전원 가능
```
